## ADDED Requirements

### Requirement: 首次掃描
系統 SHALL 對附加程序的可讀記憶體區域執行首次掃描，搜尋符合條件的數值。

#### Scenario: 精確值掃描
- **WHEN** 使用者選擇數值型別並輸入目標值，執行首次掃描
- **THEN** 系統掃描所有可讀記憶體區域，回傳所有符合該值的位址列表

#### Scenario: 未知初始值掃描
- **WHEN** 使用者選擇「未知初始值」並執行首次掃描
- **THEN** 系統記錄所有可讀記憶體區域的快照，供後續篩選使用

### Requirement: 後續篩選掃描
系統 SHALL 支援在首次掃描結果上進行後續篩選。

#### Scenario: 精確值篩選
- **WHEN** 使用者輸入新的目標值並執行後續掃描
- **THEN** 系統僅在先前結果中搜尋，回傳符合新值的位址子集

#### Scenario: 變化比較篩選
- **WHEN** 使用者選擇「增加」、「減少」、「不變」或「已改變」比較模式
- **THEN** 系統比較當前值與上次掃描快照，回傳符合條件的位址

### Requirement: 支援多種數值型別
系統 SHALL 支援以下掃描數值型別：byte (u8/i8)、2 bytes (u16/i16)、4 bytes (u32/i32)、8 bytes (u64/i64)、float (f32)、double (f64)、位元組陣列、字串 (UTF-8)。

#### Scenario: 各型別掃描
- **WHEN** 使用者選擇任一支援的數值型別
- **THEN** 掃描引擎按該型別的大小與對齊方式正確解讀記憶體資料

### Requirement: 多執行緒掃描
系統 SHALL 使用多執行緒平行掃描以提升效能。

#### Scenario: 大範圍記憶體掃描
- **WHEN** 目標程序有大量可讀記憶體區域
- **THEN** 掃描引擎將區域分配至多個執行緒平行處理，掃描完成後合併結果

### Requirement: 掃描進度回報
系統 SHALL 在掃描過程中回報進度。

#### Scenario: 顯示掃描進度
- **WHEN** 掃描正在進行
- **THEN** UI 顯示掃描進度百分比與已找到的結果數量

### Requirement: 重設掃描
系統 SHALL 允許使用者重設掃描，清除所有結果並回到初始狀態。

#### Scenario: 重設
- **WHEN** 使用者按下重設鍵
- **THEN** 清除所有掃描結果與快照，回到可執行首次掃描的狀態
